Da ein Ziel dieser Diplomarbeit die Möglichkeit ist, bereits gespielte Spiele zu analysieren, wird ein Webinterface benötigt. Dieses besteht aus drei Komponenten: Website (Frontend), Server (Backend) und Datenbank. Alle drei sind eng miteinander verbunden und arbeiten zusammen, um eine reibungslose Benutzererfahrung zu gewährleisten.

\subsection{Entwicklungsumgebung und Software-Architektur}
Die Website ist die Benutzeroberfläche, über die die Nutzer mit dem System interagieren. Sie bietet Funktionen wie die Anzeige von bereits gespielten Spielen, die Erstellung von neuen Nutzern und das Spielen neuer Spiele. 

\subsubsection{Entwicklungs-Tools}
Für die Entwicklung des Webinterfaces wurden verschiedene Tools und Technologien verwendet. Dazu gehören:
\begin{itemize}
    \item Visual Studio Code: Dieser Code-Editor wurde für die Entwicklung des Backends und des Frontends verwendet. 
    \item Nano: Wurde auf dem Server verwendet, um Dateien direkt zu bearbeiten.
    \item Git: Wurde für die Versionskontrolle und Zusammenarbeit verwendet, um den Code effizient zu verwalten und Änderungen nachzuverfolgen.
    \item Python: Die Hauptprogrammiersprache für die Backend-Entwicklung.
    \item Flask: Ein leichtgewichtiges Webframework für Python, das gut mit den anderen Komponenten zusammenarbeitet.
    \item MariaDB: Ein relationales Datenbankmanagementsystem, das zur Speicherung der Spieldaten verwendet wird.
    \item HTML/CSS/JavaScript: Für die Gestaltung und Funktionalität des Frontends.
\end{itemize}

\subsection{Realisierung der Benutzeroberfläche}\cite{W3Schools}
Die Benutzeroberfläche der Website wurde mit HTML, CSS und JavaScript gestaltet, wobei HTML für die Struktur und den Inhalt der Webseiten, CSS für das Styling und Layout, und JavaScript für die Interaktivität und dynamische Funktionen verwendet werden.


\subsubsection{Struktur}
Die Website besteht aus mehreren Seiten, wobei jede eine spezifische Funktion erfüllt:
\begin{itemize}
    \item \textbf{index.html:} Die Startseite, die eine Übersicht über die Funktionen der Website bietet.
    \item \textbf{Benutzereingabe.html:} Hier können Nutzer ihre Informationen eingeben, um ein neues Konto zu erstellen oder sich anzumelden.
    \item \textbf{schachbrett.html:} Diese Seite zeigt das Schachbrett an, auf dem die Nutzer spielen können. Das Schachbrett wurde selbsständig gestaltet, um eine individuelle und ansprechende Benutzererfahrung zu bieten.
    \item \textbf{Spielanalyse.html:} Hier können Nutzer bereits gespielte Spiele analysieren und ihre Züge überprüfen.
    \item \textbf{weiterleitung.html:} Diese Seite dient als Weiterleitung nach bestimmten Aktionen, wie z.B. nach der Anmeldung oder nach dem Speichern eines Spiels.
\end{itemize}

\subsubsection{Der Fluss der Daten}
Die Daten fließen zwischen dem Frontend und dem Backend über API-Aufrufe.
Ein Beispiel: \\
\texttt{Wenn der Nutzer auf schachbrett.html einen Zug macht, registriert ein JavaScript-Event-Listener diese Aktion und bereitet die Daten für den POST-Request an den /save\_game-Endpunkt vor. Der Server empfängt diese Daten, verarbeitet sie und speichert sie in der Datenbank. Sobald die Daten erfolgreich gespeichert wurden, \\ sendet der Server eine Bestätigung zurück an das Frontend, das dann die Benutzeroberfläche entsprechend aktualisiert, um den neuen Spielstand anzuzeigen.}

\subsection{Backend-Logik und Socket-Kommunikation}
Das Backend ist der Teil im Hintergrund der Website, der die Logik und die Funktionalität bereitstellt. Es verarbeitet die Anfragen der Nutzer, kommuniziert mit der Datenbank und stellt die benötigten Informationen bereit. Das Backend ist entscheidend für die Funktionalität der Website und sorgt dafür, dass alles reibungslos funktioniert.

Die Implementierung des Backends erfolgt in Python. 

\subsubsection{Struktur des Backends}
Die Codestruktur sieht wie folgt aus:
\begin{itemize}
    \item backend.py: \ref{lst:backend} Dies ist die Hauptdatei, die den Server startet und die Routen definiert. 
    \begin{itemize}
        \item die Module \texttt{os, socket, logging, threading, mysql.connector, flask und flask\_cors} werden importiert
        \item Es wird eine Flask-Anwendung mit CORS-Unterstützung erstellt, um Cross-Origin-Anfragen zu erlauben
        \item Debug-Logging wird für die Fehlerverfolgung konfiguriert
        \item Die Umgebungsvariablen für \texttt{Host, Benutzer, Passwort und Datenbankname} werden in der Datenbank-Konfiguration implementiert
        \item Die Funktionen sind:
        \begin{itemize}
            \item /save\_game(POST): Diese Funktion nimmt die JSON-Daten (\texttt{user\_id, pgn, gegner, ergebnis}) entgegen, verbindet sich mit der Datenbank und fügt ein neues Spiel in die Tabelle \texttt{spiele} ein. Es wird entweder Erfolg oder Fehler zurückgegeben.
            \item /get\_games/<int:user\_id>(GET): Diese Funktion holt alle Spiele für einen bestimmten Benutzer aus der Datenbank und gibt sie als JSON zurück. 
            \item receive\_file(): Die Funktion startet eine TCP-Server auf IP 10.10.10.100, Port 5001 und wartet auf Verbindungen. Sie empfängt Daten und speichert sie in \texttt{received\_game.pgn}. Sie läuft in einer Endlosschleife. 
            \item in der main-Funktion wird \texttt{receive\_file()} in einem separaten Thread (im Daemon-Modus) gestartet, damit er beim Beenden der App stoppt und die Flask-App wird auf IP 10.10.10.100 am Port 5000 gestartet. 
            \item im Anhang befindet sich der Code unter \texttt{backend.py}.
        \end{itemize}
    \end{itemize}

    \item \_\_init\_\_.py: Flask sucht gezielt nach dieser Datei, um den root-Pfad zu finden. Sie kann jedoch leer bleiben.
    \item wsgi.py: Dieser Code lädt die Flask-App und startet sie mit \texttt{app-run()}, wenn sie direkt ausgeführt wird. Er dient als Wrapper, um die App über WSGI-Server zu deployen, anstatt direkt \texttt{backend.py} zu starten.
    \begin{scriptsize}
        \begin{lstlisting}[style=Python, language=Python, captionpos=b, stepnumber=5, caption = {wsgi.py}]
            from backend import app

            if __name__ == "__main__":
                app.run()
        \end{lstlisting}
    \end{scriptsize}
    \item nginxNextmove: Hier ist nginx konfiguriert. Der Server lauscht auf Port 80 (http) unter der IP 10.10.10.100. Anfragen an die Hauptseite (/) werden zu den richtigen Ordner auf der Festplatte geleitet (\texttt{/home/username/website/frontend}), um dort die \texttt{index.html} anzuzeigen \cite{NginxGuide}. Alle Anfragen, die mit \texttt{/api/} beginnen, werden an das Python-Flask weitergeleitet. 

    \begin{scriptsize}
        \begin{lstlisting}[style=Python, language=Python, captionpos=b, stepnumber=5, caption = {nginxNextmove}]
    server {
        listen 80;
        server_name nextmove.htlec.org 10.10.10.100;

        root /home/username/website/frontend;
        index index.html;

        location / {
            try_files $uri $uri/ =404;
        }

        location /api/ {
            proxy_pass http://127.0.0.1:5000/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
        \end{lstlisting}
    \end{scriptsize}
    \item nextmove.service: Diese Datei stellt sicher, dass UWSGI und Flask die Daten für die Datenbank finden \cite{uWSGI}. Das Einloggen startet mit \texttt{After=network.target} automatisch und mit \texttt{Restart=always} wird der Backend-Server sofort neu gestartet, falls er abstürzt. Unter \texttt{Environment} werden die Zugangsdaten für die Datenbank direkt an die App übergeben und der Befehl \texttt{ExecStart} nutzt \texttt{uwsgi}, um die Flask-App auf Port 5000 bereitzustellen. Über den Pfad \texttt{/home/username/venv/bin} wird sichergestellt, dass die Python-Bibliotheken aus der venv-Umgebung genutzt werden. Dieser Code ist das 'Gegenstück' zu der nginxNextmove-Datei: 
    \begin{itemize}
        \item nginxNextmove nimmt Anfragen aus dem Internet an
        \item Wenn die Anfrage an \texttt{/api/} geht, leitet nginxNextmove sie an 127.0.0.1:5000 weiter.
        \item nextmove.service sorgt dafür, dass auf Port 5000 die Flask-App via uWSGI durchgehend bereit steht. 
    \end{itemize}
    \begin{scriptsize}
        \begin{lstlisting}[style=Python, language=Python, captionpos=b, stepnumber=5, caption = {nextmove.service}]
    [Unit]
Description=A uWSGI instance to serve the Flask nextmove backend
After=network.target

[Service]
User=username
Group=www-data
WorkingDirectory=/home/username/website/backend

# Deine Datenbank-Geheimnisse m  ssen hier auch rein!
Environment="DB_HOST=localhost"
Environment="DB_USER=username"
Environment="DB_PASSWORD=BeispielPasswort"
Environment="DB_NAME=NextMove"
Environment="PATH=/home/username/venv/bin"

# Hier ist der komplette Befehl ohne K  rzungen:
ExecStart=/home/username/venv/bin/uwsgi --http 0.0.0.0:5000 --chdir /home/username/website/backend --module wsgi:app --master ->

Restart=always

[Install]
WantedBy=multi-user.target
        \end{lstlisting}
    \end{scriptsize}
\end{itemize}
