\subsection{Rotary encoder als Eingabegerät}
Zur Auswahl des passenden Eingabegeräts, kamen einige Varianten auf:
\begin{itemize}
	\item Das Steuern über vier Buttons, nach dem D-pad (Abkürzung für directional pad) Prinzip
	\item Über einen Joystick wie auf einem Konsolencontroller
	\item Über einen Rotary encoder, wie er zum Beispiel in Autos verbaut ist
\end{itemize}

Die schlussendliche Wahl fiel dann auf den Rotary encoder, da er eine gute Mischung aus modernität und 
funktionalität bietet. Bei dem Modell, welches im Schachbrett verbaut wurde, handelt es sich um den KY-040. 
Dieser ist der mit unter am weitesten verbreiteste seiner Art und die Einbindung mit dem Raspberry Pi 5 
ist dank der gpiozero library auch praktisch.

\subsubsection{Funktionsweise eines Rotary encoders}
Die Art, wie solch ein Inkrementalgeber funktionert, ist für viele auf den ersten Blick vielleicht nicht 
ganz verständlich und wird wahrscheinlich mit der Funktionweise eines Potentiometers verwechselt. 
Allerdings handelt es sich hier um einen kodierten Drehschalter. Der Hauptunterschied, welcher bei 
Bedienung schnell auffällt ist, dass sich dieses Bauteil, im Vergleich zum klassischen Potentiometer, 
\glqq unendlich lange\grqq{} drehen lässt. Jeder Drehpunkt kann taktil erfühlt werden.
Ein Inkrementalgeber hat zwei Ausgangssignale, A und B, welche bei Drehung ein periodisches Rechtecksignal 
ausgeben.

\begin{lstlisting}[style=Python, language=Python, captionpos=b, stepnumber=5, caption = {Erstes Beispiel für die Einbindung eines Rotary encoders}]
import threading

from gpiozero import RotaryEncoder, Button


class Encoder:
    def __init__(self):
        self.encoder = RotaryEncoder(13, 19, wrap = True,
									max_steps=100,
									bounce_time=0.05)

        self.button = Button(16, pull_up = True, 
							bounce_time = 0.05,
							hold_time = 2)

        self.done = threading.Event()

        self.button.when_released = self.handle_button  
        self.encoder.when_rotated = self.handle_rotate


    def handle_rotate(self):
        print(self.encoder.steps)


    def handle_button(self):
        print("Button released")

	
if __name__ == "__main__":
    encoder = Encoder()
    encoder.done.wait()

\subsection{Entwicklung eines Menüs}
Ein benutzerfreundliches Menü, welches zu Beginn des einschaltens erscheint, ist essenziell für ein gutes 
Spielerlebnis. Eine erste Überlegung, wie die Struktur dieses Spielmenüs aussehen soll, war wie folgt 
aufgebaut:
Wird das Schachbrett gestartet, wird auf einem Display, welches sich auf dem Schachbrett befindet, 
angezeigt, dass sich der Benutzer zwischen  \glqq Offlinespiel\grqq{} oder \glqq Onlinespiel\grqq{} 
entscheiden kann. 


\subsubsection{\glqq Offlinespiel\grqq{}}
Wird ersteres gewählt, so erscheint ein Untermenü. Hier besteht die Wahl zwischen 
\glqq Spieler vs. Spieler\grqq{} und \glqq Spieler vs. COM\grqq{}.
\glqq Spieler vs. Spieler\glqq{} steht dafür, dass der Benutzer gegen einen Gegenüber Schach spielen kann, 
ganz ohne Einmischung eines Schachcomputers. Im Falle der Entscheidung auf diese Möglichkeit, startet das 
Spiel sofort. Die Auswahl \glqq Spieler vs. COM\grqq{} hingegen erlaubt es, noch vor dem Beginn des Spiels 
zwischen eine von drei Schwierigkeitsstufen — Leicht, Mittel oder Schwer — zu wählen, welche der 
Schachcomputer haben soll. Die Stufen unterscheiden sich davon, wie lange sich der Schachcomputer Zeit 
lässt, um einen optimalen Zug zu berechnen.
Nachdem eine Schwierigkeitsstufe gewählt wurde, geht nun das Spiel los und der Nutzer kann spielen.


\subsubsection{\glqq Onlinespiel\grqq{}}
Wird sich anstelle des Einzelspiels für das Onlinespiel entschieden, erscheint ein Untermenü, in dem der 
Benutzer aufgefordert wird, sich entwerder einzulogen oder einen neuen account zu registrieren. Ersteres 
geht davon aus, dass der Benutzer schon über ein Bentzerkonto verfügt und deshalb nur noch den NFC-Chip 
an das Lesegerät auf dem Schachbrett legen soll, damit der Benutzer in der Datenbank verglichen werden kann. 
Kann kein bereits bestehender Nutzer dem NFC-Chip zugeordnet werden, so wird der Nutzer infolgedessen 
aufgefordert, sich zu registrieren. Beim registrieren wird ein neuer Eintrag in der Datenbank erstellt, 
und der Benutzer kann fortfahren.
Im Falle des Onlinespiels kann der Spieler nur gegen einen Schachcomputer spielen und im Anschluss daran 
seine Partie auf der Website analysieren.


\subsubsection{Verknüpfen von Menü und Encoder}
Die Menüstruktur, welche zuvor beschrieben wurde, muss jetzt mit dem Encoder so verknüpft werden, dass der 
Benutzer mit diesem durch die verschiedenen Optionen navigieren kann. Es muss also so programmiert werden, 
dass die Drehung des Encoders die Auswahl im Menü ändert und das Klicken des Buttons die Auswahl bestätigt.
Die Hürde hierbei ist, dass sich die Menüstruktur in einem Baumdiagramm befindet, welches sich je nach 
Auswahl verändert. Es muss also so programmiert werden, dass die Auswahlmöglichkeiten im Menü immer an 
die aktuelle Position im Baum angepasst werden.

\begin{lstlisting}[style=Python, language=Python, captionpos=b, stepnumber=5, caption = {Erstes Beispiel für die Einbindung eines Rotary encoders}]
import time
import threading

from luma.oled.device import sh1106
from luma.core.render import canvas
from luma.core.interface.serial import i2c
from gpiozero import RotaryEncoder, Button



class Menu():
    def __init__(self):
        self.ai_diff = 1
        self.menu_state = 0
        self.selected_index = 0

        # Initialize rotary encoder and button
        self.encoder = RotaryEncoder(11, 9, wrap = True,
									max_steps=100,
									bounce_time=0.05)

        self.button = Button(10, pull_up = True,
							bounce_time = 0.05,
							hold_time = 2)
        
		self.done = threading.Event()

        self.serial = i2c(port = 1, address = 0x3C)
        self.device = sh1106(self.serial)

        self.button.when_released = self.handle_button
        self.button.when_held = self.stop_script
        self.encoder.when_rotated = self.handle_rotate


    def handle_rotate(self):
        #print(self.encoder.steps)

        match self.menu_state:
            case 0:  # Splash Screen
                    pass
            case 1:  # Main Menu
                if self.encoder.steps % 2 == 0:
                    self.selected_index = 0  # Singleplayer
                elif self.encoder.steps % 2 == 1:
                    self.selected_index = 1  # Multiplayer
            case 2:  # Login Screen
                if self.encoder.steps == 0:
                    self.selected_index = 0  # Login
                elif self.encoder.steps == 1:
                    self.selected_index = 1  # Register
            case 3:  # AI Difficulty Selection
                if self.encoder.steps == 0:
                    self.ai_diff = 1  # Easy
                elif self.encoder.steps == 1:
                    self.ai_diff = 2  # Medium
                elif self.encoder.steps == 2:
                    self.ai_diff = 3  # Hard
            case 4:  # Game Screen
                    pass
            
    def handle_button(self):
        match self.menu_state:
            case 0:  # Splash Screen
                pass
            case 1:  # Main Menu
                if self.selected_index == 0:
                    self.menu_state = 2  # Go to Login Screen
                elif self.selected_index == 1:
                    self.menu_state = 4  # Start Multiplayer Game
            case 2:  # Login Screen
                if self.selected_index == 0:
                    self.menu_state = 4  # Singleplayer (Login)
                elif self.selected_index == 1:
                    self.menu_state = 4  # Singleplayer (Guest)
            case 3:  # AI Difficulty Selection
                self.menu_state = 4  # Start Singleplayer (AI set)
            case 4:  # Game Screen
                pass


    def stop_script(self):
        self.done.set()


    def draw_menu(self):
        if self.device:
            with canvas(self.device) as draw:
                match self.menu_state:
                    case 0:  # Splash Screen
                        draw.text((35, 25), "Nextmove", fill = "white")
                        draw.rectangle(self.device.bounding_box, outline="white")
                        self.menu_state = 1  # Go to Main Menu
                    case 1:  # Main Menu
                        options = ["Einzelspieler", "Mehrspieler"]
                        draw.text((40, 0), "-- Home --", fill = "white")
                        for i, option in enumerate(options):
                            if i == self.selected_index:
                                draw.text((20, 15 + i*15), "> " + option, fill = "white")
                            else:
                                draw.text((20, 15 + i*15), option, fill = "white")
                    case 2:  # Login Screen
                        draw.text((25, 0), "-- Login --", fill = "white")
                        options = ["Login", "Register"]
                        for i, option in enumerate(options):
                            if i == self.selected_index:
                                draw.text((20, 15 + i*15), "> " + option, fill = "white")
                            else:
                                draw.text((20, 15 + i*15), option, fill = "white")

                    case 3:  # AI Difficulty Selection
                        draw.text((15, 0), "-- AI Difficulty --", fill = "white")
                        options = ["Easy", "Medium", "Hard"]
                        for i, option in enumerate(options):
                            if i == self.ai_diff - 1:
                                draw.text((20, 15 + i*15), "> " + option, fill = "white")
                            else:
                                draw.text((20, 15 + i*15), option, fill = "white")

                    case 4:  # Game Screen
                        draw.text((20, 0), "-- Game On --", fill = "white")
        else:
            print("Nextmove")
            time.sleep(2)


if __name__ == "__main__":
    menu = Menu()
    try:
        # Main loop
        while not menu.done.is_set():
            menu.draw_menu()
            time.sleep(0.05)
    except KeyboardInterrupt:
        pass
\end{lstlisting}
\end{scriptsize}

Der Aufbau dieses Codes ist so gestaltet, dass das wechseln in die verschiedenen Untermenüs mit Hilfe von 
\glqq match-case\grqq{} Blöcken umgesetzt wird.

\begin{itemize}
	\item case 0: Splash Screen
	\item case 1: Main Menu
	\item case 2: Login Screen
	\item case 3: AI Difficulty Selection
	\item case 4: Game Screen
\end{itemize}
Gestartet wird mit einem Splash Screen, welcher den Namen des Schachbretts anzeigt. 
Nach einer kurzen Zeitspanne wechselt das Menü automatisch zum Hauptmenü, in dem der Benutzer zwischen 
Einzelspieler und Mehrspieler wählen kann.

Unter case 2: \glqq Login Screen\grqq{} wird die Auswahl zwischen Login und Register getroffen. 
In diesem ersten Aufbau wird, wenn sich sowohl für Login als auch für Register entschieden wird, direkt 
auf menu state 4: \glqq Game Screen\grqq{} gewechselt wird. In einem späteren Schritt soll 
hier natürlich noch die Funktionalität des Logins bzw. der Registrierung implementiert werden, damit der 
Wechsel zum Spielbildschirm erst dann passiert, wenn der Login bzw. die Registrierung erfolgreich war.
